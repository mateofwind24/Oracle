# Oracle Report 요구사항 명세서

문서 버전: v1.0  
작성일: 2026-06-25  
대상 프로젝트: Oracle Report

---

## 1. 적용 범위

Oracle Report는 로컬 장치에서 얼굴 캡처, 관상 보조 분석, 사주/만세력 조회, 개인 리포트, 궁합 리포트를 생성하는 Raspberry Pi 기반 웹 애플리케이션이다.

이 문서의 요구사항은 다음 범위에 적용한다.

- Flask 기반 웹 UI
- `run.sh`, `build.sh` 실행 스크립트
- llama.cpp OpenAI-compatible API 기반 로컬 LLM 호출
- SQLite 만세력 DB와 관상/추천 정보 DB
- 개인 리포트와 두 사람 궁합 리포트 생성 흐름
- 프롬프트 템플릿과 LLM 출력 검증
- 테스트와 실행 재현 자료

## 3. 실행 및 설정 요구사항

| ID | 요구사항 | 우선순위 | 인수 기준 |
|---|---|---:|---|
| RUN-001 | `./run.sh build`는 기본적으로 프로젝트 루트의 `.venv`에 Python 환경을 설치해야 한다. | P0 | 별도 옵션 없이 의존성 설치와 패키지 설치가 완료된다. |
| RUN-002 | `./run.sh`는 웹 UI를 실행해야 한다. | P0 | 기본 포트 `8501`에서 Flask UI에 접속할 수 있다. |
| RUN-003 | 실행 스크립트는 로컬 llama.cpp 서버 실행을 지원해야 한다. | P0 | `ORACLE_START_LLAMA_SERVER=1`일 때 모델 경로가 있으면 서버 실행을 시도한다. |
| RUN-004 | 실행 스크립트는 `debug`와 `release` 모드를 구분해야 한다. | P0 | debug는 산출물을 저장하고, release는 임시 산출물을 실행 후 정리한다. |
| RUN-005 | 시스템은 프롬프트 확인 모드와 LLM 결과 확인 모드를 제공해야 한다. | P0 | `prompt`, `llm`, `prompt-run` 명령이 각각 동작한다. |
| RUN-006 | 이름, 생년월일, 출생시간, 성별은 CLI 파라미터로 입력할 수 있어야 한다. | P0 | 프롬프트 생성과 LLM 실행 명령에서 동일한 입력 형식을 사용한다. |
| RUN-007 | 자주 바뀌는 설정은 환경 변수와 config 파일에서 관리해야 한다. | P0 | `.env.example`, `configs/raspberry_pi.env`, `configs/prompts.json`로 주요 설정을 확인할 수 있다. |
